EVE Features

EVE is a front-end data processiong language. EVE stands for Extract Validate Export. Hoever eve & Eve & EVE are synonim names. Eve is imperative multi-paradigm, staticly typed with type inference.

Targt Features:

#FeatureDescription
1 readable use familiar keywords
2 effective focus on domain specific jobs
3 safe use static type system
4 modular has reusable scripts and modules
5 versatile can connect to multiple databases
Design: EVE has a classic design inspired from Ada & Java. It uses classic keywords and structures. Therefore learning EVE is more easy if you know other programming languages from Ada family. EVE is not a curly bracket language and is much more different than C++, Java, Rust, Go or Swift but has some similarity with Python.

Readable

EVE is designed to be learn by reading code examples. Be syntax is verbose and explicit. We believe that our design choices will make EVE code easier to read and comprehend:

  • Use familiar English keywords, with no abbreviation;
  • Use declarative statements based on noun keywords;
  • Use a sugestive keyword to finish a block statement;
  • Use only ASCII operators and delimiters not Unicode symbols;
  • Use comprehensive data literals for constants and collections;
  • Use uppercase and lowercase to separate data types from variables;
  • Use a sigil/prefix for system declarations to reduce name collisions;

The result of these choices is demonstrate below with an example. You can observe different comment notations enable different colors. Syntax colorizer is able to identify the keywords and highlighted every statement. This can not be achieved using a language that has curly brackets used as delimiters for blocks.

# Fibonacci series demonstrate
driver fibonacci:

import $eve_lib.console.(*);

** declare Fibonacci routine
function fib(n:Integer) => (y:Integer):
  if (n == 1) or (n == 0) then
    let y := 1; -- first value
  else
    let y := fib(n-1) + fib(n-2);
  done;
return;

process main:
   ** local variable for result
   new r: Integer;

   ** call fib rule using argument 5
   let r := fib(5);

   ** print the result to console
   print r;
return;

Effective

EVE is aiming to be effective and feasible. This means EVE is design to improve developer productivity and get the job done as quickly as possible. Effective does not necessarily imply efficiency or higher performance. EVE is designed to perform one task very well one time or a limited number of times by using a time scheduler. EVE is not design for high performance computing.

  • Use gradual typing;
  • Use type inference;
  • Use separation of concerns;
  • Enable exclusive database connectivity;
  • Enable model first application design;
  • Enable generative (macro) programming;

Safe

EVE is designed to be a safe language. For this we avoid using Null pointers. EVE is using boxed values that are actually references but we initialize all the variables with zero value. We avoid having Null references. EVE is and Object Oriented and declarative language. We believe the next principles will help us to design a safe language:

  1. We favor safety over performance,
  2. It is better to be productive than efficient,
  3. Implicit values are better than wrong values,
  4. A good error message worth more than gold,
  5. If the system is not ready it should tell you,
  6. If a process fail it should report the errors.

Modular

EVE applications are usually small, based on single source file. However EVE enable usage of multiple files for separation of concerns. There are 3 kind of modules in a large EVE application as you can see in the following diagram:

    EVE modular design

  1. One main script called "driver"
  2. Multiple secondary scripts called "aspects"
  3. Multiple library modules called simple "modules"

Drivers and aspects are executable while modules are reusable but not executable. Aspects are executed from driver process and are individual process. Once an aspect is resolved it is removed from memory, unlike modules that are persistent until the driver ends.

Versatile

EVE is a domain specific language but it looks and feel like a general purpose language. We do not do graphics but we manipulate data in any form. This can be text data but also row bit data. So we can read and write images and video streams. Next design choices will make EVE a versatile language:

  1. EVE variables and parameters can be declared using type hint, tupe inference or gradual typing. This improve readability and reduce clutter;
  2. EVE use implicit data types that are static but enable also variant type. The variant type can support a number of other data types that can be specified. So one variant is more like a pointer than regular variables.
  3. EVE has processes that can support input/output parameters. A process can run in a separate thread. However a process can not be used in expressions.
  4. EVE has functions. However the functions are like named expressions. A function must be simple and can return a single result and can not have side-effects.
  5. Variables and parameters are automatically initialized with zero value when there is no explicit initialization. Also, if a parameter has explicit initial value it becomes optional.
  6. EVE has two assignment operators: "::" and ":=". First is called "clone" second is called "assign" value. Developers can control what assignment operator to use depending on the algorithm requirements.
  7. In a subroutine, primitive type parameters are transferred by reference, but basic type parameters are immutable, protected by the compiler.

Classic

EVE is using a familiar syntax. We try to avoid being disruptive. If you know Java or C++ you will be able to learn EVE in less then one week. Also if you know Ada or Pascal, you will find EVE most similar.

  1. EVE is using classic control flow statements: {decision, selection, repetition, error handlers} that you already know from other languages.
  2. EVE is using ASCII operators, and keyword based operators but not Unicode. This enable fast code input required to write large business oriented scripts.
  3. EVE has user defined data types (classes). EVE is a hybrid language it has suport for object oriented programming OOP.
  4. EVE is familiar with SQL, HTML and JSON. To deal with many SQL dialects, EVE have specific drivers that enable direct interaction with major databases.
